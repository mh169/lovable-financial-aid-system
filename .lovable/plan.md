

# خطة إصلاح وتجهيز نظام الدعم المالي v5

## نتائج الاختبار

### ✅ ما يعمل بشكل صحيح (تم اختباره عبر API):
- **إنشاء طلب جديد** — يعمل ويعيد الاستمارة مع رقم ID صحيح
- **تسجيل الدخول** — يعمل (admin/admin123) ورفض كلمة المرور الخاطئة
- **إدارة المستخدمين** — 5 مستخدمين افتراضيين (admin, user1, user2, viewer1, viewer2)
- **إنشاء/عرض المشاريع** — يعمل (اسم مشروع + بند + مبلغ)
- **إنشاء مسير صرف/إخلاء/تحصيل** — يعمل ويعيد أرقام المسيرات
- **تحديث الاستمارة** — يعمل
- **حذف الاستمارة** — يعمل (HTTP 204)
- **عرض المرضى والجهات** — يعمل
- **أعمدة الطباعة** — صحيحة (14 عمود صرف، 12 إخلاء، 13 تحصيل)
- **رقم الاستمارة في الطباعة** — يظهر بشكل صحيح ("رقم الاستمارة" وليس "رقم استمارة المساعدة")
- **صور رأس الاستمارة** — موجودة في client/public/
- **نظام الصلاحيات** — مشاهد لا يرى "طلب جديد"، فقط المدير يرى "الصلاحيات"
- **تكرار ترويسة الجدول عند الطباعة** — `thead { display: table-header-group; }`

### ❌ خطأ يجب إصلاحه:
1. **`getNextFormNumber` يُرجع `"000NaN"`** — لأن `parseInt("F-002")` يُرجع `NaN`. الدالة تفترض أن `form_number` رقم صرف، لكن القيم مثل `"F-002"` تحتوي على حرف. يجب استخراج الرقم فقط أو الاعتماد على MAX(id).

## الإصلاح المطلوب

### الخطوة 1: إصلاح `getNextFormNumber` في `server/storage.ts`
تغيير الدالة لتتعامل مع أرقام استمارات تحتوي على حروف:
```typescript
async getNextFormNumber(): Promise<string> {
  const db = await this.db();
  const r = queryOne(db, "SELECT MAX(CAST(REPLACE(REPLACE(form_number,'F-',''),'F','') AS INTEGER)) as mx FROM financial_forms WHERE form_number IS NOT NULL");
  const current = r?.mx || 0;
  return String(current + 1).padStart(6, '0');
}
```

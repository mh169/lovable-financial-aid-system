

# خطة اختبار نظام الدعم المالي v7

## ما سيتم تنفيذه

### 1. فك الأرشيف وتشغيل السيرفر
- فك `financial-support-system-v7.tar.gz` في `/tmp`
- تثبيت المكتبات (`npm install`)
- تشغيل السيرفر على منفذ 3000

### 2. اختبار إنشاء طلب جديد + طباعة داخلية
- تسجيل دخول كـ admin
- إنشاء طلب جديد عبر API مع بيانات كاملة
- التحقق من أن الاستجابة تحتوي على `formNumber` صحيح
- فحص كود `printTemplates.ts` للتأكد من:
  - ظهور رقم الاستمارة في قالب الطباعة الداخلية
  - وجود QR Code في الطباعة الداخلية (يحتوي على: اسم المستلم، رقمه، جهة التسليم، المبلغ، رقم الاستمارة، الرقم الطبي، اسم المريض، الاحتياج)
  - عدم وجود QR في الطباعة الخارجية

### 3. اختبار صفحة الصلاحيات
- فحص API `PUT /api/users/:id` للتأكد من قبول `canAdd, canEdit, canDelete, canView`
- اختبار تغيير صلاحيات مستخدم عبر API
- التحقق من أن الصلاحيات تُطبق (مثلاً: مستخدم بدون `canAdd` لا يستطيع إنشاء طلب)

### 4. اختبار تسجيل اسم المستخدم عند التعديل/الحذف
- فحص كود `routes.ts` و`storage.ts` للتأكد من أن حقل `updated_by` يُملأ عند:
  - إنشاء طلب جديد
  - تعديل طلب
  - حذف طلب
- إذا لم يكن موجوداً → إصلاحه

### 5. إصلاح أي أخطاء مكتشفة وإعادة التغليف
- إصلاح أي مشاكل
- إنشاء `financial-support-system-v7.1.tar.gz` إذا تم تعديل شيء

## المهام المطلوبة (8 مهام)

### المهمة 1: إنشاء ملف مرجعي شامل للنظام
إنشاء ملف نصي (`system-specification.md`) يحتوي على:
- كل ميزة ووظيفة ومكون في النظام
- هيكل قاعدة البيانات (جداول + أعمدة)
- واجهات API كاملة
- هيكل الملفات (server/client/shared)
- تنسيقات الطباعة (أعمدة كل مسير)
- نظام الصلاحيات والأدوار
- تدفق العمل (workflow) لكل عملية

**لا يغير النظام — ملف توثيقي فقط**

### المهمة 2: إضافة متابعة حالة الصرف + متابعة حالة التحصيل
حالياً توجد متابعة حالة الإخلاء فقط. سأضيف نفس النمط لـ:
- **متابعة حالة الصرف**: تتبع حالة كل استمارة في مسير الصرف (مصروف/لم يصرف/معلق)
- **متابعة حالة التحصيل**: تتبع حالة كل استمارة في مسير التحصيل (محصّل/لم يحصّل/معلق)
- إضافة تبويبات جديدة في الواجهة + API endpoints + أعمدة حالة في DB

**يضيف ميزات جديدة — لا يلغي شيء**

### المهمة 3: إضافة QR Code في طباعة الاستمارة الداخلية
- استخدام مكتبة `qrcode` (محلية، بدون إنترنت)
- QR يحتوي على: اسم المستلم، رقم المستلم، جهة التسليم، المبلغ كتابة، رقم الاستمارة، الرقم الطبي، اسم المريض، الاحتياج
- يظهر فقط في نموذج الطباعة الداخلية
- **الطباعة الخارجية تبقى كما هي بدون QR**

**يضيف ميزة — لا يغير تنسيق الطباعة الداخلية/الخارجية الحالي**

### المهمة 4: تحسين تنسيقات النظام (UI فقط)
- تحسين واجهة المستخدم (الألوان، التخطيط، الأيقونات) بشكل احترافي
- التأكد من أن كل المكتبات محلية (لا CDN)
- **استثناء صريح: تنسيقات طباعة الاستمارة الداخلية والخارجية تبقى كما هي تماماً بدون أي تعديل**

**يحسن المظهر فقط — لا يمس وظائف أو تنسيقات الطباعة**

### المهمة 5: زر "طلب جديد من طلب سابق" في السجلات
- إضافة زر في كل صف بجدول السجلات
- عند الضغط: فتح تبويب "طلب جديد" مع تعبئة البيانات من الطلب السابق (كطلب جديد بدون ID)
- يسهّل إنشاء طلبات متكررة لنفس المريض

**يضيف ميزة — لا يلغي شيء**

### المهمة 6: تطوير نظام الصلاحيات
- إضافة صفحة إدارة صلاحيات متقدمة للمدير
- تحديد لكل مستخدم: إضافة ✅/❌، تعديل ✅/❌، حذف ✅/❌، مشاهدة ✅/❌
- حفظ الصلاحيات في جدول `users` أو جدول منفصل
- تطبيق الصلاحيات في الواجهة + API

**يطوّر ميزة موجودة — الأدوار الحالية (admin/user/viewer) تبقى كقاعدة + صلاحيات تفصيلية**

### المهمة 7: اختبار شامل
- تشغيل السيرفر واختبار كل ميزة عبر API
- إنشاء طلب + حفظ + طباعة داخلية/خارجية
- التأكد من رقم الاستمارة + QR
- اختبار المسيرات الثلاثة + متابعة الحالات
- اختبار الصلاحيات بكل الأدوار
- التأكد من عدم فقد أي ميزة

### المهمة 8: تجهيز الأرشيف النهائي v7
- حذف بيانات الاختبار
- إنشاء `financial-support-system-v7.tar.gz` + ملف المواصفات

| المهمة | تغيير/إلغاء ميزة؟ |
|--------|-------------------|
| ملف مرجعي | ❌ لا |
| متابعة صرف/تحصيل | ❌ إضافة فقط |

| تحسين UI | ❌ مظهر فقط (الطباعة لا تتأثر) |
| طلب من سابق | ❌ إضافة فقط |
| صلاحيات تفصيلية | ⚠️ تطوير (لا إلغاء — توسيع النظام الحالي) |
| اختبار + تغليف | ❌ لا |

**لا يتم إلغاء أي ميزة موجودة. كل التعديلات إضافية أو تحسينية.**

## التفاصيل التقنية

### الملفات المتأثرة:
```text
server/
  ├── storage.ts          — إضافة دوال متابعة الصرف/التحصيل + صلاحيات تفصيلية
  ├── init-db.ts           — إضافة أعمدة حالة + جدول صلاحيات
  └── routes.ts            — إضافة API endpoints جديدة

client/src/pages/
  ├── VoucherTracking.tsx   — صفحة جديدة: متابعة حالة الصرف
  ├── CollectionTracking.tsx — صفحة جديدة: متابعة حالة التحصيل
  ├── Records.tsx           — إضافة زر "طلب جديد من سابق"
  ├── NewRequest.tsx        — قبول بيانات أولية من السجلات + QR في الطباعة الداخلية
  └── Permissions.tsx       — تطوير صفحة الصلاحيات

client/src/components/layout/
  └── Sidebar.tsx           — إضافة تبويبات متابعة الصرف/التحصيل

shared/
  └── routes.ts             — إضافة أنواع جديدة

package.json               — إضافة مكتبة qrcode
```

### مكتبات مطلوبة (محلية):
- `qrcode` — لتوليد QR Code بدون إنترنت
